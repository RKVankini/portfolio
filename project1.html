Highly Available AWS Multi‑Region 3‑Tier Architecture

End‑to‑end multi‑region 3‑tier infrastructure on AWS with failover using Route 53, CloudFront, ALB, ASG, and Multi‑AZ RDS (read‑replica), built with a warm‑standby DR strategy.

Project Overview

This project demonstrates deployment of a production‑grade 3‑tier architecture across two AWS regions (us‑east‑1 primary & us‑west‑2 DR). It includes VPC and subnet design, ALBs, Auto Scaling Groups, Security Groups, RDS Multi‑AZ with cross‑region read‑replica, CloudFront failover, Route 53 health checks, and WAF protection. The implementation follows AWS best practices for high availability and disaster recovery.

Architecture Diagram

High‑Level Steps

  1. VPC Setup: Public + private subnets across 2 AZs in each region, NAT gateways, route tables.
  2. Security Layer: SGs for Bastion, ALBs, app servers, and RDS with least‑privilege rules.
  3. Compute Layer: EC2 golden images, AMIs, and launch templates for frontend/backend.
  4. Load Balancing: Frontend & backend ALBs with HTTPS listeners and ACM certs.
  5. RDS Layer: MySQL Multi‑AZ primary + cross‑region read‑replica for DR.
  6. Failover Setup: Route 53 failover policies + CloudFront origin groups.
  7. Security: AWS WAF applied to CloudFront distribution.
  8. Testing: Simulated regional outage to validate DR behavior.

Tools & Services

  • AWS: VPC, EC2, ALB, ASG, RDS, Route 53, CloudFront, WAF, ACM
  • OS & Config: Ubuntu, Apache, Node.js, PM2
  • Networking & Security: Subnets, NAT, IGW, SGs, DNS failover

Responsibilities & Achievements

  • Designed multi‑region architecture with warm‑standby DR.
  • Configured failover routing using Route 53 health checks.
  • Implemented CloudFront origin failover for global latency optimization.
  • Automated backend/frontend setup using launch templates.
  • Built RDS architecture supporting cross‑region failover.
  • Documented full deployment steps and DR testing results.
