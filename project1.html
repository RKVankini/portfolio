AWS 3‑Tier Application Deploy (Warm‑Standby DR)

Role: Infrastructure & DevOps · Tools: AWS (VPC, ALB, ASG, RDS, Route53, CloudFront, WAF) · Deliverables: infra build + DR playbook

This project demonstrates a production‑grade 3‑tier architecture on AWS with a warm‑standby disaster recovery strategy across two regions (us‑east‑1 primary, us‑west‑2 DR). The repo contains step‑by‑step instructions for VPC/subnets, NAT/IGW, security groups, ALBs, target groups, EC2 + AMI/launch templates, Auto Scaling Groups, RDS (primary + read‑replica), Route 53 failover, CloudFront origin groups, and AWS WAF configuration.

Open Full Project PDF

Quick Architecture

Three logical layers: Presentation (ALB + CloudFront + static frontend), Application (ASG-backed backend servers behind ALB), Data (RDS multi‑AZ with read‑replica in DR region). Health checks + Route 53 failover and CloudFront origin groups provide automated failover for the frontend/backends while database failover uses read‑replica promotion.

Key Steps (high level)

  • Provision VPC with public + private subnets (8 subnets total across AZs).
  • Create NAT Gateways, IGW, route tables, and enable DNS hostnames.
  • Configure Security Groups for bastion, ALBs, app servers and RDS (least privilege).
  • Launch temp EC2 golden images, configure app & frontend, create AMIs and Launch Templates.
  • Create Target Groups, ALBs (frontend & backend) and attach HTTPS listeners with ACM certs.
  • Set up RDS (MySQL) primary in us‑east‑1, enable backups and create read‑replica in us‑west‑2.
  • Use Route 53 (failover records + health checks) and CloudFront origin groups for cross‑region failover.
  • Add WAF Web ACL and rules for CloudFront distribution.
  • Test failover by simulating regional outage; document cleanup steps to remove billable resources.
