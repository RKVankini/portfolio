Multi-Cloud IaC with Terraform

Reusable, modular Terraform modules for provisioning infrastructure across AWS, GCP, and Azure.

Project Overview

Built modular Terraform code to provision networking, compute, and managed database resources across AWS, GCP, and Azure. Implemented remote state, environment workspaces, and secrets management for secure multi-environment deployments.

Key Responsibilities & Features

  • Designed modules for VPC / VNet, subnets, routing, and security groups.
  • Implemented remote state using AWS S3 + DynamoDB for locking.
  • Created environment-specific workspaces (dev/test/prod) and parameterized deployments with variables, locals, and outputs.
  • Used `count`, `for_each`, and `dynamic` blocks to keep modules DRY and reusable.
  • Automated instance bootstrap with Terraform provisioners and integrated Ansible for post-provisioning configuration.
  • Managed secrets via AWS Secrets Manager (or equivalent for other clouds) and avoided hard-coded credentials.
  • Integrated Terraform with GitHub for version-controlled IaC and documented runbooks in the repo README.
  • Performed cost-aware testing by destroying test environments after validation.

Tech Stack

  • Terraform (modules, workspaces, remote state)
  • Clouds: AWS, GCP, Azure
  • Secrets: AWS Secrets Manager / Cloud IAM
  • Automation: Ansible, Bash
